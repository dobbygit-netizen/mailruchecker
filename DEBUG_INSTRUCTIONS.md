# Инструкции по отладке Mail.ru Checker

## Как проверить работу расширения и найти проблему

### Шаг 1: Перезагрузите расширение

1. Откройте `chrome://extensions/`
2. Найдите "Mail.ru Checker"
3. Нажмите кнопку "Обновить" (круговая стрелка)

### Шаг 2: Откройте консоль разработчика

#### Для Background Script (основные логи):
1. На странице `chrome://extensions/` найдите расширение
2. Включите "Режим разработчика" (Developer mode) в правом верхнем углу
3. Под расширением найдите ссылку "service worker" или "Проверить представления: фоновая страница"
4. Нажмите на нее - откроется консоль разработчика

#### Для Offscreen Document (логи проверки):
1. Откройте всплывающее окно расширения (кликните по иконке)
2. Нажмите F12 для открытия консоли
3. Нажмите кнопку "Проверить сейчас" в расширении
4. В консоли появятся детальные логи

### Шаг 3: Проверьте авторизацию в Mail.ru

Расширение использует ваши текущие cookies браузера для доступа к Mail.ru. Убедитесь:

1. Откройте https://e.mail.ru в обычной вкладке
2. Войдите в свой аккаунт, если ещё не вошли
3. После успешного входа попробуйте снова проверить почту в расширении

### Шаг 4: Проверьте настройки расширения

1. Кликните по иконке расширения
2. Нажмите "Настройки"
3. Убедитесь, что ваш email добавлен в список аккаунтов
4. Формат должен быть: `example@mail.ru` (без пароля для этого расширения)

### Шаг 5: Анализ логов

После нажатия "Проверить сейчас" в консоли должны появиться логи:

#### Успешная проверка через API:
```
Offscreen: проверка аккаунта example@mail.ru
API ответ - статус: 200, ok: true
Полный ответ API: {...}
✓ Найдено непрочитанных через API: 5
```

#### Если API недоступен (переход на iframe):
```
API ответ - статус: 401, ok: false
API недоступен или требует авторизации
Проверка через iframe...
Iframe загружен
Найдено непрочитанных писем через селектор "...": 5
```

#### Если iframe не работает (переход на fetch):
```
Ошибка парсинга iframe: ...
Пробуем альтернативный метод через fetch...
Проверка через fetch...
HTML получен, длина: 123456
Найдено непрочитанных через fetch: 5
```

### Что означают ошибки:

1. **"API недоступен или требует авторизации"** - Нормально, расширение переключится на другой метод
2. **"Ошибка парсинга iframe: SecurityError"** - CORS блокировка, расширение использует fetch
3. **"Не удалось найти количество непрочитанных писем"** - Mail.ru изменил структуру страницы

### Шаг 6: Отправьте логи для анализа

Если проблема не решена:

1. Скопируйте все логи из консоли
2. Сделайте скриншот страницы с логами
3. Откройте страницу https://e.mail.ru/inbox/ и сделайте скриншот (чтобы было видно количество непрочитанных)
4. Предоставьте эту информацию для дальнейшего анализа

### Временное решение

Если расширение не работает, попробуйте:

1. Выйти из аккаунта Mail.ru
2. Войти заново
3. Перезагрузить расширение
4. Попробовать снова

## Технические детали

### Методы проверки (в порядке приоритета):

1. **API метод** - `https://e.mail.ru/api/v1/folders/counters`
   - Быстрый, надёжный
   - Требует авторизации через cookies

2. **Iframe метод** - загрузка страницы в невидимом iframe
   - Медленнее, но работает если API недоступен
   - Парсит DOM страницы

3. **Fetch метод** - прямой HTTP запрос
   - Последний резервный вариант
   - Парсит HTML через regex

### Необходимые разрешения:

- `storage` - сохранение настроек
- `notifications` - уведомления
- `offscreen` - для безопасного парсинга
- `https://e.mail.ru/*` - доступ к Mail.ru
